Tell me the difference between a process and a thread.
  - Process: A process is an independent and self-contained unit of execution that consists of its own memory space, program code, system resources, and a single thread of execution. Processes are isolated from each other
  - Thread: A thread is a smaller unit of execution within a process. Threads share the same space and resources as the parent process, allowing multiple threads to work together
  
What are common issues in handling interrupts?
  - Priority and nesting: high-priority interrupts can preempt lower-priority interrupts potentially delaying low ISRs to be delayed.
  - Interrupt latency: the delay between an interrupt is triggered and the ISR begins executing can be problematic for real-time constraints.
  - Shared resources: Without proper synchronization mechanisms, sumulatneous access by multple interrupts can lead to race conditions or data corruption.
  - Reentrancy: Some ISRs can be reentrant, meaning they can be interrupted by the same or another instance of the interrupt.
  - Interrupt Overloading: Handling too many interrupts simultaneously or frequently can overload the processor,
  - Interrupt Chaining: An interrupt can trigger another interrupt. If not managed properly, this can lead to unintended recursion or excessive interrupt chaining.
  - Priority Inversion: Lower-priority interrupts might hold resources needed by higher-priority ISRs, causing priority inversion
